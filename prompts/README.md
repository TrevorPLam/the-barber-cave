> **Project Context**: The Barber Cave
> Stack: Next.js 16 (App Router), React 19, TypeScript strict, Tailwind 4, Zod, Vitest, Playwright
> Priorities: Performance, Security (CSP hardened), Accessibility (WCAG 2.2 AA), Enterprise patterns

## How to Use This Library

For maximum reliability:

1. Always PLAN first
2. Review output
3. Ask for risk clarification
4. Approve implementation
5. Run performance audit

7. Run regression test expansion

This is how you orchestrate AI like a CTO.

## Usage Guidelines

All prompts assume **PLAN mode first** unless explicitly stated.

### When to Use Each Prompt

- **Daily/Weekly**: Individual audit prompts for specific areas
- **Monthly**: Comprehensive audits across multiple categories
- **Quarterly**: Master system audit for complete health check

### Best Practices

1. **Start with PLAN mode** - Always review before implementing
2. **Document findings** - Keep track of identified issues
3. **Prioritize by risk** - Address critical issues first
4. **Verify fixes** - Run regression tests after changes
5. **Monitor performance** - Use Lighthouse to track improvements

This library is designed to help maintain high-quality, enterprise-grade code while enabling rapid, AI-driven development.
